2. DESCRIPTION DU LIVRABLE
================================================================================

Contenu du livrable :

A. MODULE D'AUTHENTIFICATION
   - Inscription utilisateur avec validation par email
   - Connexion (login/logout) avec JWT (access token + refresh token)
   - Réinitialisation de mot de passe (forgot password / reset password)
   - Vérification d'email
   - Gestion automatique du rafraîchissement des tokens
   - Redirection automatique selon le rôle utilisateur

B. GESTION DES UTILISATEURS ET RÔLES
   - Système multi-rôles :
     * Superadmin (administration globale)
     * OF Admin (administrateur d'organisme de formation)
     * Gestionnaire
     * Formateur interne
     * Créateur de contenu
     * Formateur indépendant
     * Apprenant
     * Tuteur
   - Profils utilisateurs personnalisables (photo, bio, coordonnées)
   - Validation des formateurs par le superadmin
   - Gestion du statut utilisateur (actif/inactif)
   - Système d'alertes utilisateur (validation, rejet)

C. GESTION DES ORGANISATIONS DE FORMATION (OF)
   - Création d'organismes de formation par le superadmin
   - Configuration des OF (informations légales, agrément, logo)
   - Sous-domaines personnalisés (slug.learneezy.com)
   - Gestion des utilisateurs affiliés à un OF
   - Système d'abonnements (Starter, Gold, Premium)
   - Attribution de tokens IA par OF

D. GESTION DES COURS
   - Création de cours (CRUD complet)
   - Structure hiérarchique : Cours > Modules > Leçons > Quiz
   - Upload de médias (images, vidéos)
   - Gestion du statut des cours (draft, published)
   - Inscription aux cours (enrollment)
   - Statistiques des cours (nombre d'inscrits, revenus, scores moyens)
   - Suivi de progression par apprenant

E. SYSTÈME DE CATALOGUES
   - Catalogues globaux et spécifiques aux OF
   - Attribution de catalogues aux apprenants
   - Filtrage des cours selon l'abonnement

F. SYSTÈME DE TOKENS
   - Consultation du solde de tokens
   - Achat de tokens (intégration Stripe simulée)
   - Historique des transactions

G. STRUCTURE DE LA BASE DE DONNÉES
   - Base PostgreSQL avec les tables principales :
     * users (utilisateurs)
     * organizations (organismes de formation)
     * subscriptions (abonnements)
     * catalogues (catalogues de formation)
     * token_transactions (transactions de tokens)
   - Cours stockés dans MongoDB (structure flexible)

H. API REST FASTAPI
   - Documentation OpenAPI complète disponible
   - Endpoints d'authentification (/api/auth/*)
   - Endpoints de gestion des cours (/api/courses/*)
   - Endpoints de gestion des OF (/api/organizations/*)
   - Endpoints d'abonnements (/api/subscriptions-and-catalogues/*)
   - Endpoints de tokens (/api/tokens/*)
   - Endpoints d'administration (/api/auth/superadmin/*)

I. INTERFACES UTILISATEUR
   - Page d'accueil publique (hero, témoignages, partenaires)
   - Pages d'inscription et de connexion
   - Formulaire d'inscription apprenant complet
   - Dashboards différenciés par rôle :
     * Dashboard Superadmin
     * Dashboard OF Admin
     * Dashboard Apprenant
     * Dashboard Tuteur
     * Dashboard Formateur indépendant
     * Dashboard Gestionnaire
     * Dashboard Formateur interne
     * Dashboard Créateur de contenu
   - Système de navigation avec sidebars personnalisées
   - Mode sombre/clair
   - Gestion des CGV et politique de confidentialité (modals)

Objectifs du livrable : 
Fournir la première version fonctionnelle du socle technique de la plateforme
Learneezy avec les modules de base permettant l'inscription, l'authentification
et la gestion des cours par différents types d'utilisateurs.

4. ARCHITECTURE TECHNIQUE
================================================================================

• Frontend :
  - Architecture basée sur React avec routing multi-pages
  - Hooks personnalisés pour l'authentification (useFastAPIAuth, useAuthForm)
  - Client API centralisé (fastapi-client.ts)
  - Gestion automatique des tokens JWT avec refresh
  - Components réutilisables et modulaires
  - Design system cohérent (index.css + tailwind.config.ts)

• Backend :
  - API RESTful avec FastAPI
  - Documentation OpenAPI auto-générée
  - Authentification OAuth2 avec JWT
  - Validation des données avec Pydantic
  - Gestion des emails (vérification, reset password, notifications)
  - Système de rôles et permissions

• Sécurité :
  - Tokens JWT avec expiration
  - Refresh tokens pour renouvellement automatique
  - Validation email obligatoire
  - Routes protégées par rôle
  - Hashage des mots de passe
  - Protection CORS
